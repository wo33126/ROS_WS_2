# 固定路线模块使用说明

本文档说明如何使用固定路线执行模块，让底盘按预设路线自动移动，并且方便后续改路线。

---

## 1. 模块说明

新增节点：

- `fixed_route_runner_node`

作用：

- 从 YAML 读取路线段列表
- 按顺序持续发布 `/cmd_vel`
- 每一段都使用统一结构：`vx`、`vy`、`omega`、`duration`
- 路线逻辑独立于运动学和 CAN 接口，后续改路线不需要改底层控制代码

执行链路：

- `fixed_route_runner_node` -> `/cmd_vel`
- `planner_node` -> `/motor_velocity_cmd`
- `can_interface_node` -> STM32 自定义 CAN 帧

---

## 2. 路线文件位置

默认路线文件：

- [robot_bringup/config/fixed_route.yaml](robot_bringup/config/fixed_route.yaml)

你通常只需要修改这个文件，不需要改 C++ 源码。

---
## 3. YAML 格式

示例：
auto_start: true
stop_between_segments: false

    duration: 2.0
  - name: move_right
    vx: 0.00
    vy: 0.25
    omega: 0.00
    duration: 2.0

  - name: rotate_ccw
    vx: 0.00
    vy: 0.00
    omega: 0.60
    duration: 2.0
```

参数含义：

- `publish_rate_hz`：固定路线节点发布 `/cmd_vel` 的频率
- `start_delay_sec`：启动后等待多久再开始执行
- `loop_route`：执行完最后一段后是否循环
- `auto_start`：启动后是否自动开始
- `stop_between_segments`：每一段切换时是否先发布一次零速度
- `publish_zero_when_paused`：暂停或外部挂起时是否持续发布零速度

每个 `route` 段包含：

- `name`：路线段名称，仅用于日志显示
- `vx`：前后方向线速度，单位 m/s
- `vy`：左右方向线速度，单位 m/s
- `omega`：旋转角速度，单位 rad/s
- `duration`：该段持续时间，单位 s

## 7. 与其他进程的兼容接口

为了便于你在路线执行过程中插入其他功能，固定路线模块已经预留了运行时控制接口。

### 7.1 控制输入接口


```bash
/fixed_route/control   std_msgs/String
/fixed_route/hold      std_msgs/Bool
```
- `start`：从当前段开始执行
- `restart`：从第 0 段重新开始
- `pause`：暂停路线
```bash
rostopic pub /fixed_route/control std_msgs/String "data: 'pause'" -1
rostopic pub /fixed_route/control std_msgs/String "data: 'resume'" -1
rostopic pub /fixed_route/control std_msgs/String "data: 'restart'" -1
```

`/fixed_route/hold` 适合其他业务进程临时抢占控制权：

- `true`：外部挂起路线输出
- `false`：释放挂起，路线继续

示例：

```bash
rostopic pub /fixed_route/hold std_msgs/Bool "data: true" -1
rostopic pub /fixed_route/hold std_msgs/Bool "data: false" -1
```

推荐用法：

- 路线模块负责默认巡航
- 其他进程一旦需要接管，先发 `/fixed_route/hold=true`
- 外部功能完成后，再发 `/fixed_route/hold=false`

这样不需要停节点，也不需要改底层规划器。

### 7.2 状态输出接口

状态话题：

```bash
/fixed_route/status                std_msgs/String
/fixed_route/current_segment       std_msgs/String
/fixed_route/is_running            std_msgs/Bool
/fixed_route/is_paused             std_msgs/Bool
/fixed_route/external_hold_active  std_msgs/Bool
```

用途：

- 其他进程可订阅当前段名称和状态字符串
- 上层任务管理器可根据 `is_running` / `is_paused` / `external_hold_active` 做流程编排

示例：

```bash
rostopic echo /fixed_route/status
rostopic echo /fixed_route/current_segment
rostopic echo /fixed_route/is_running
```

## 8. 设计说明

当前设计是模块化的：

- 固定路线模块只负责“发布什么车体速度”
- `planner_node` 只负责“把车体速度换算成四轮 RPM”
- `can_interface_node` 只负责“把 RPM 通过 STM32 协议发出去”

这样后续如果你要扩展：

- 多条路线切换
- 循环路线
- 条件触发下一段
- 通过服务或话题动态切换路线
- 中途插入机械臂动作、视觉定位或避障流程
- 上层状态机临时抢占 `/cmd_vel`

都可以只改固定路线模块，而不需要动底层 CAN 和运动学代码。
