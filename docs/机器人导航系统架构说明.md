# 机器人导航系统架构说明

> 本文档全面说明本工作空间中所有功能包的作用、依赖关系、数据流向及整体架构设计思路。
> 系统目标：**全向移动底盘 + 视觉识别 + 距离传感器 + 机械臂协作，完成药品自动配送任务。**

---

## 1. 系统概述

本系统是一个**医院病房药品自动配送机器人**，核心能力包括：

| 能力 | 说明 |
|------|------|
| 🚗 **全向移动** | 四轮麦克纳姆轮底盘，支持平移、旋转等全向运动 |
| 👁️ **视觉识别** | 摄像头识别护士台二维码（解析任务）和床头条形码（验证药品） |
| 🗺️ **路径导航** | 预设路径点跟踪，闭环 PID 控制到达目标位置 |
| 🎯 **精确对准** | VL53L1X 激光测距传感器进行终点距离微调 |
| 🦾 **药品操作** | 步进电机机械臂 + 双舵机夹爪，自动取放药品 |
| 📦 **药箱管理** | 独立药箱 CAN 控制，按任务开启对应药箱 |

---

## 2. 架构总览

系统采用**分层架构**，从底层硬件到顶层任务编排。存在**两条核心控制链路**：

```
┌──────────────────────────────────────────────────────────────────────────┐
│                        🔷 链路 B: 任务编排（横向）                        │
│                                                                          │
│  ┌──────────────┐    ┌──────────────────┐    ┌─────────────────────┐     │
│  │ 摄像头图像    │───▶│ barcode_qr       │───▶│ mission_controller   │     │
│  │ /usb_cam     │    │ _detector        │    │ （任务状态机）        │     │
│  └──────────────┘    └──────────────────┘    └──────┬──────┬───────┘     │
│                                                     │      │             │
│                             ┌───────────────────────┘      │             │
│                             │ 服务调用                      │ 服务调用     │
│                             ▼                               ▼             │
│               ┌─────────────────┐             ┌─────────────────────┐    │
│               │ path_manager    │             │ fine_tuning         │    │
│               │ （选路径+发点）  │             │ （终点距离微调）      │    │
│               └───────┬─────────┘             └──────────┬──────────┘    │
│                       │                                  │               │
│                       ▼                                  ▼               │
│               ┌─────────────────┐             ┌─────────────────────┐    │
│               │ path_tracker    │             │ arm_and_gripper     │    │
│               │ （逐点跟踪）     │             │ （机械臂放药动作）    │    │
│               └───────┬─────────┘             └─────────────────────┘    │
│                       │                                                  │
└───────────────────────┼──────────────────────────────────────────────────┘
                        │ /cmd_vel
                        ▼
┌──────────────────────────────────────────────────────────────────────────┐
│                        🔷 链路 A: 底盘控制（纵向）                        │
│                                                                          │
│  /cmd_vel ──▶ cmd_vel_mux ──▶ planner_node ──▶ /motor_velocity_cmd      │
│               (多源合并)       (Twist→RPM)         (Float32MultiArray)    │
│                                                          │               │
│                                                          ▼               │
│                                                    can_interface_node    │
│                                                    (RPM→CAN帧→STM32)     │
│                                                          │               │
│                                                          ▼               │
│                                                    STM32 → 四轮电机      │
│                                                          │               │
│  反馈: 电机状态 ←── CAN ←── STM32 ←── 电机编码器                        │
│                                                          │               │
│                                                          ▼               │
│                              base_odometry_node ──▶ /odom (TF)           │
│                              (RPM→里程计)                                 │
└──────────────────────────────────────────────────────────────────────────┘
```

### 分层对照表

| 层号 | 层名 | 功能包 | 核心职责 |
|:----:|------|--------|----------|
| 1 | 🏗️ 硬件通信层 | `can_motor_interface` | 底盘电机 CAN 帧收发 |
| 2 | 🧮 运动规划层 | `motion_planner` | 运动学解算、里程计、多源速度合并 |
| 3 | 🦾 机械臂层 | `arm_interface` | 机械臂关节 CAN 通信（可选） |
| 4 | 🗺️ 路径管理层 | `path_manager` | 路径加载、选择、逐点发布 |
| 5 | 🎯 路径跟踪层 | `path_tracker`（在 path_manager 内） | PID 闭环跟踪路径点 |
| 6 | 🔍 终点微调层 | `fine_tuning` | VL53L1X 激光测距精确对准 |
| 6.5 | 📡 传感器驱动层 | `vl53l1x_driver`（在 robot_navigation 内）🆕 | VL53L1X I2C 驱动 + 滤波 |
| 6.8 | 🛡️ 安全检测层 | `distance_safety`（在 robot_navigation 内）🆕 | 距离安全检测，自动制动/后退 |
| 7 | ⚙️ 舵机控制层 | `servo_controller`（在 arm_and_gripper 内） | pigpio PWM 舵机控制 |
| 8 | 🧠 业务编排层 | `arm_and_gripper` + `robot_navigation` | 药品摆放动作 + 任务状态机 |

> **注意**：`robot_navigation` 横跨所有层，作为"大脑"通过服务调用协调各层。`robot_bringup` 是纯启动/参数包。

## 3. 两条核心控制链路详解

### 3.1 链路 A: 底盘运动控制链路

这是**最基础的控制链路**，负责将任何来源的速度指令转化为电机转动：

```
多源 /cmd_vel 输入:
  ├── distance_safety  (安全制动，最高优先级) 🆕
  ├── 键盘遥控 (teleop_twist_keyboard)
  ├── path_tracker (路径跟踪输出)
  ├── fine_tuning (微调输出)
  └── fixed_route_runner (固定路线)

         │
         ▼
    cmd_vel_mux_node          优先级: safety > external > teleop > fixed_route
         │                        ↑
         │               distance_safety_node 发布到 /cmd_vel_safety 🆕
         │ 选中的 /cmd_vel (Twist: vx, vy, omega)
         ▼
    planner_node              运动学解算（麦克纳姆轮逆向运动学）
         │                    Twist → 四轮各自 RPM
         │
         ▼
    /motor_velocity_cmd       Float32MultiArray [rpm1, rpm2, rpm3, rpm4]
         │
         ▼
    can_interface_node        RPM → CAN 扩展帧 → STM32 电机驱动器
         │
         ▼
    STM32 → 四台闭环步进电机  (ZDT Y42 第二代)
         │
         │ 反馈: 编码器 → STM32 → CAN → can_interface_node
         ▼
    /motor_state              Float32MultiArray [rpm1, rpm2, rpm3, rpm4] (实际)
         │
         ▼
    base_odometry_node        运动学正解: 各轮 RPM → 底盘速度 → 积分位置
         │
         ▼
    /odom                     nav_msgs/Odometry (位姿 + 速度)
    TF: odom → base_footprint
```

### 3.2 链路 B: 任务编排控制链路

这是**完整的任务执行链路**，由 `mission_controller` 状态机驱动：

```
┌─────────────────────────────────────────────────────────────────────┐
│                      任务状态机 (mission_controller)                 │
│                                                                     │
│  IDLE ──▶ GOTO_NURSE ──▶ SCAN_QR ──▶ GOTO_BED_A ──▶ POSITION_A    │
│                                          │                          │
│                                          ▼                          │
│                            SCAN_BARCODE_A → OPEN_BOX_A → PLACE_A    │
│                                          │                          │
│                                          ▼                          │
│                            GOTO_BED_B → ... → PLACE_B → RETURN_HOME │
│                                                                     │
│  全局超时: 180 秒 → /mission_timeout → /stop_all                    │
└─────────────────────────────────────────────────────────────────────┘

每个状态的执行方式:
  GOTO_*      → 调用 /select_path 服务，path_tracker 自动跟踪
  POSITION_*  → 调用 /fine_tuning/start 服务，精确对准
  SCAN_*      → barcode_qr_detector 识别二维码/条形码
  OPEN_BOX_*  → open_medicine_box 通过 CAN 开启对应药箱
  PLACE_*     → 调用 /arm_place_medicine 服务，机械臂执行放药
```

**状态机与底层包的协调方式**：

| 状态 | 调用的服务 | 等待的信号 |
|------|-----------|------------|
| `GOTO_NURSE` | `/select_path "nurse_station"` | `/path_finished` |
| `SCAN_QR` | 无（被动接收） | `/qr_result` 非空 |
| `GOTO_BED_A/B` | `/select_path "bed_X"` | `/path_finished` |
| `POSITION_IN_CIRCLE_A/B` | `/fine_tuning/start` | `/fine_tuning_done` |
| `SCAN_BARCODE_A/B` | 无（被动接收） | `/barcode_bedX` 非空 |
| `OPEN_BOX_A/B` | `/open_medicine_box` | 服务返回成功 |
| `PLACE_MEDICINE_A/B` | `/arm_place_medicine` | 服务返回成功 |
| `RETURN_HOME` | `/select_path "home"` | 进入起始区多边形 + 静止 5s |

### 4.2 `motion_planner` — 运动规划

| 属性 | 内容 |
|------|------|
| **层级** | 第2层 · 运动规划层 |
| **包含节点** | `cmd_vel_mux_node`、`planner_node`、`base_odometry_node`、`fixed_route_runner_node`、`twist_test_node` |
| **职责** | 多源速度合并、麦克纳姆轮运动学解算、里程计计算、固定路线执行 |

#### 子节点详解

| 节点 | 功能 | 输入 | 输出 |
|------|------|------|------|
| `cmd_vel_mux_node` | 多源 `/cmd_vel` 优先级合并 | 多路 `/cmd_vel` 话题 | 选中的 `/cmd_vel` |
| `planner_node` | **运动学逆解**：底盘 Twist → 各轮 RPM | `/cmd_vel` (Twist) | `/motor_velocity_cmd` |
| `base_odometry_node` | **运动学正解**：各轮 RPM → 里程计 | `/motor_state` | `/odom`、TF |
| `fixed_route_runner_node` | 从 YAML 读取固定路线序列并逐段执行 | `fixed_route.yaml` | `/cmd_vel` (定时序列) |
| `twist_test_node` | 测试工具：发布 Twist 指令 | 无 | `/cmd_vel` |

**麦克纳姆轮运动学**（实际代码实现）：

逆向运动学（`planner_node`，Twist → 各轮线速度）：
```
v_front  =  vx + ω × L      // 前轮（0°方向）
v_right  =  vy + ω × L      // 右轮（90°方向）
v_back   = -vx + ω × L      // 后轮（180°方向）
v_left   = -vy + ω × L      // 左轮（270°方向）

RPM_i = v_i × 60 / (2π × wheel_radius)
```

正向运动学（`base_odometry_node`，各轮 RPM → 底盘速度）：
```
vx  = 0.5 × (v_front - v_back)
vy  = 0.5 × (v_right - v_left)
ω   = (v_front + v_right + v_back + v_left) / (4 × L)
```

其中 `L = wheel_base`（轮心到车体中心距离），正逆解对应一致。

| **节点** | `path_manager_node` |
| **职责** | 从 YAML 加载多条命名路径，运行时支持按名称选择、动态加载，按固定频率发布当前路径点 |

```yaml
    points:
      - {x: 0.0, y: 0.0, tolerance: 0.05}
      - {x: 2.0, y: 1.0, tolerance: 0.03}
```

### 4.5 `path_tracker` — 路径跟踪器

| 属性 | 内容 |
|------|------|
| **层级** | 第5层 · 路径跟踪层 |
| **节点** | `path_tracker_node`（位于 `path_manager` 包内） |
| **职责** | 订阅路径点和里程计，使用比例控制器驱动机器人逐点到达目标 |

| 方向 | 名称 | 类型 | 含义 |
|------|------|------|------|
| 订阅 ⬇️ | `/path_points` | `PathPoint` | 当前目标路径点 |
| 订阅 ⬇️ | `/odom` | `Odometry` | 里程计（当前位置） |
| 发布 ⬆️ | `/cmd_vel_external` | `Twist` | 速度控制指令 |
| 发布 ⬆️ | `/path_finished` | `Bool` (Latch) | 路径终点到达信号 |

**控制算法**：

- **默认模式** (`use_turn_then_move=false`)：同时控制线速度 + 角速度，平滑逼近目标
  - `v = kp_linear × distance_error`
  - `ω = kp_angular × heading_error`
- **分步模式** (`use_turn_then_move=true`)：先原地旋转对准方向，再直线前进

### 4.6 `fine_tuning` — 终点距离微调

| 属性 | 内容 |
|------|------|
| **层级** | 第6层 · 终点微调层 |
| **节点** | `fine_tuning_node` |
| **职责** | 路径跟踪到达终点后，使用 VL53L1X 激光测距传感器进行 PID 精确距离对准 |

| 方向 | 名称 | 类型 | 含义 |
|------|------|------|------|
| 服务 🟢 | `/fine_tuning/start` | `Trigger` | 启动微调 |
| 订阅 ⬇️ | `/vl53l1x_distance` | `Float32` | 激光测距读数 |
| 发布 ⬆️ | `/cmd_vel_external` | `Twist` | 微调速度指令 |
| 发布 ⬆️ | `/fine_tuning_done` | `Bool` (Latch) | 微调完成 |

**微调过程**：

```
启动 → 读取距离 → PID计算速度 → 下发 /cmd_vel → 底盘移动
  ↑                                                      |
  └──────────── 循环直至距离误差 < tolerance ─────────────┘
                                                         ↓
                                              发布 /fine_tuning_done
```

### 4.7 `vl53l1x_driver` — VL53L1X 距离传感器驱动 🆕

| 属性 | 内容 |
|------|------|
| **层级** | 第6.5层 · 传感器驱动层 |
| **节点** | `vl53l1x_driver_node`（位于 `robot_navigation` 包内） |
| **职责** | 通过 I2C 直接读取 VL53L1X ToF 激光测距传感器，发布距离数据及安全信号 |

| 方向 | 名称 | 类型 | 含义 |
|------|------|------|------|
| 发布 ⬆️ | `/vl53l1x_distance` | `Float32` | 滤波后的距离值 (mm) |
| 发布 ⬆️ | `/vl53l1x/status` | `String` | 传感器状态 (OK/ERROR) |
| 发布 ⬆️ | `/distance_safety_warning` | `Bool` | 距离低于安全阈值 |
| 发布 ⬆️ | `/distance_safety_stop` | `Bool` | 距离低于紧急阈值 |
| 服务 🟢 | `/vl53l1x/reset` | `Trigger` | 重置传感器 |
| 服务 🟢 | `/vl53l1x/calibrate` | `Trigger` | 偏移校准 |

**主要特性**：
- I2C 通信（默认 `/dev/i2c-1`, 地址 `0x29`）
- 中值滤波 + EMA 双重滤波
- 三级安全检测：WARNING（减速）→ STOP（停止）→ REVERSE（后退）
- 传感器数据超时自动报警

### 4.8 `distance_safety` — 距离安全检测 🆕

| 属性 | 内容 |
|------|------|
| **层级** | 第6.8层 · 安全检测层 |
| **节点** | `distance_safety_node`（位于 `robot_navigation` 包内） |
| **职责** | 底盘移动中实时监测前方距离，通过 `/cmd_vel_safety` 通道自动制动 |

| 方向 | 名称 | 类型 | 含义 |
|------|------|------|------|
| 订阅 ⬇️ | `/vl53l1x_distance` | `Float32` | 前方距离 (mm) |
| 订阅 ⬇️ | `/cmd_vel` | `Twist` | 当前底盘速度（判断方向） |
| 发布 ⬆️ | `/cmd_vel_safety` | `Twist` | 安全制动指令（mux 最高优先级） |
| 发布 ⬆️ | `/distance_safety/state` | `String` | 安全状态: CLEAR/WARNING/STOP/REVERSE/TIMEOUT |

**安全策略**：

```
距离 > safety × 1.5   → CLEAR     (不干预)
距离 < safety × 1.5   → WARNING   (减速至 30%)
距离 < safety          → STOP      (发布零速度)
距离 < critical        → REVERSE   (低速后退避险)
传感器超时             → TIMEOUT   (安全停止)
```

**与 cmd_vel_mux 集成**：
```
cmd_vel_mux 优先级: safety > external > teleop > fixed_route
                        ↑
              distance_safety_node 发布到 /cmd_vel_safety
```

### 4.9 `servo_controller` — 舵机控制器

| 属性 | 内容 |
|------|------|
| **层级** | 第7层 · 舵机控制层 |
| **节点** | `servo_controller_node`（位于 `arm_and_gripper` 包内） |
| **职责** | 通过 pigpio 库输出 PWM 信号控制两个舵机（夹爪开合） |

| 方向 | 名称 | 类型 | 含义 |
|------|------|------|------|
| 服务 🟢 | `/servo_command` | `ServoCommand` | 舵机角度/动作命令 |

**前置条件**：`sudo pigpiod` 已在后台运行。

### 4.10 `arm_and_gripper` — 药品摆放控制

| 属性 | 内容 |
|------|------|
| **层级** | 第8层 · 业务逻辑层 |
| **节点** | `arm_and_gripper_node` |
| **职责** | 协调机械臂步进电机（CAN2）和两个舵机，执行药品放置动作序列 |

| 方向 | 名称 | 类型 | 含义 |
|------|------|------|------|
| 服务 🟢 | `/arm_place_medicine` | `PlaceMedicine` | 触发药品摆放动作 |
| 订阅 ⬇️ | `/fine_tuning_done` | `Bool` | 微调完成后自动触发摆放 |

**药品摆放动作序列**：

```
① 舵机1张开 (释放夹爪1)
     ↓ 等待
② 舵机2张开 (释放夹爪2)
     ↓ 等待
③ 机械臂旋转到位 (CAN2 控制步进电机)
     ↓
④ 复位 (机械臂归位 + 舵机复位)
```

## 6. 启动流程

### 6.1 硬件前置条件

```bash
# ① CAN0（底盘电机，必须）
sudo ip link set can0 up type can bitrate 1000000

# ② CAN1（机械臂电机，如需使用）
sudo ip link set can1 up type can bitrate 1000000

# ③ pigpio 守护进程（舵机控制，如需使用）
sudo pigpiod
```

### 6.2 一键完整启动（推荐）

```bash
# 完整导航 + 任务系统
roslaunch robot_navigation mission_complete.launch

# 仅底盘运动（不含任务编排）
roslaunch robot_navigation complete_navigation.launch

# 带固定路线
roslaunch robot_navigation complete_navigation.launch enable_fixed_route:=true

# 带键盘遥控
roslaunch robot_navigation complete_navigation.launch enable_teleop:=true
```

### 6.3 分步启动（调试用）

从底层到顶层依次启动：

```bash
# 第1步: 底盘 CAN 通信
roslaunch can_motor_interface can_interface.launch

# 第2步: 运动规划（含 mux、planner、odometry）
roslaunch motion_planner planner.launch

# 第3步: 机械臂 CAN（可选）
roslaunch arm_interface arm.launch

# 第4步: 路径管理
roslaunch path_manager path_manager.launch

# 第5步: 路径跟踪
roslaunch path_manager path_tracker.launch

# 第6步: 终点微调
roslaunch fine_tuning fine_tuning.launch

# 第7步: 舵机控制
roslaunch arm_and_gripper servo_controller.launch

# 第8步: 药品摆放
roslaunch arm_and_gripper arm_and_gripper.launch

# 第9步: 任务编排（视觉 + 状态机）
roslaunch robot_navigation mission_complete.launch
```

## 9. 典型工作流程

### 9.1 完整药品配送流程

```
时间轴 →

T0: 系统上电，所有节点就绪，等待启动信号
│
T1: 按下物理按钮 → /start_signal
│
T2: mission_controller 进入 GOTO_NURSE
│   → 调用 /select_path "nurse_station"
│   → path_tracker 闭环控制底盘到达护士台
│
T3: 到达护士台 → SCAN_QR
│   → barcode_qr_detector 识别二维码: "11" (1床1号箱)
│
T4: GOTO_BED_A (1床)
│   → 调用 /select_path "bed_1"
│   → path_tracker 导航到1床附近
│
T5: 到达1床 → POSITION_IN_CIRCLE_A
│   → 调用 /fine_tuning/start
│   → VL53L1X 精确对准到目标距离
│
T6: SCAN_BARCODE_A
│   → 扫描床头药品条形码，验证药品正确
│
T7: OPEN_BOX_A
│   → CAN 指令开启1号药箱
│
T8: PLACE_MEDICINE_A
│   → 舵机1张开 → 舵机2张开 → 机械臂旋转 → 复位
│
T9: VOICE_A → 语音播报"请取药"
│
T10: GOTO_BED_B (3床) → 重复 T5-T9
│
T11: RETURN_HOME
│   → 调用 /select_path "home"
│   → 导航回起始区
│
T12: HOME_CHECK
│   → 检测进入起始区多边形 + 静止5秒
│
T13: STOP → /mission_finished → 🎉 任务完成
```

### 9.2 纯底盘控制流程（不含任务系统）

```
路径YAML → path_manager → PathPoint[] → path_tracker → /cmd_vel
                                                           ↓
                             (teleop/fixed_route) → cmd_vel_mux (合并)

  path_tracker → /path_finished → fine_tuning → /cmd_vel (微调)

```

## 10. 故障排查

| 症状 | 可能原因 | 排查方法 |
|------|----------|----------|
| 电机不转 | CAN0 未 UP | `ip link show can0` |
| 电机不转 | 电机未使能/回零 | 检查 `/motor_status_flags` |
| 舵机不响应 | pigpiod 未运行 | `sudo pigpiod` |
| path_tracker 不工作 | 无 `/odom` 数据 | 检查 `base_odometry_node` 和 `/motor_state` |
| fine_tuning 卡住 | VL53L1X 无数据 | 检查传感器连接和话题 `/vl53l1x_distance` |
| 机械臂不动 | CAN1 未 UP | `ip link show can1` |
| 二维码识别失败 | 光线不足/摄像头未连接 | 检查 `/usb_cam/image_raw` |
| launch 报错 | 未编译 | `catkin build` 或 `catkin_make` |
| 任务超时 | 导航路径过长 | 调整 `mission_params.yaml` 中超时时间 |
